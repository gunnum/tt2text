T2T System Tree

报告生产不是即时问答，而是一棵持续沉淀的分析树

素材进入 T2T 后，应该自动完成解析和标准分析；稳定、有 schema、可复用的结论要沉淀入库，临时探索只在页面展示。这个脑图用树状结构重新梳理底层系统。

主干流程

不是点一次 LLM，而是入库后进入标准流水线。

采集/上传App、TikTok、文章、Sensor Tower、评论、截图等进入系统。
原始素材入库保留可追溯原文、文件、链接、截图、CSV。
结构化解析抽取字段、路径、指标、转写、翻译、评论行。
LLM 标准分析按固定 schema 生成可复用中间结论。
结论沉淀稳定结论入库，带来源、prompt 版本和模型版本。
报告调用报告从素材层和分析层调用证据，不重复临时分析。
核心原则：入库后的标准分析不应该只是网页上的一次性展示；只要结论稳定、可复用、能追溯来源，就应该成为系统资产。

沉淀策略

不是所有 LLM 输出都该入库，要按复用价值分层。

原始素材层

必须入库

视频、转写、文章正文、CSV、评论、截图路径、来源链接。它们是所有判断的证据底座。

标准分析层

应该入库

固定 schema 的 LLM 结论要入库：hook、主题、痛点、风险、国家分工、文章观点等，并保留来源和版本。

临时探索层

只展示

一次性追问、口吻润色、没有证据引用的临时想法，不污染数据库，除非人工确认转为标准结论。

下一步建设

先把分析结果沉淀规则定下来，再开发入口。

1. 定标准分析 schema

写需求

为视频、文章、评论、Sensor Tower、体验材料分别定义分析字段、来源引用和模型版本。

2. 建分析结果表

写需求

把标准分析结论和原始素材关联起来，允许人工确认、覆盖、废弃。

3. 写产品体验 SOP

SOP

先让人按固定流程补第一手体验材料，再决定哪些字段产品化。

4. 做报告项目面板

写需求

按目标 app 显示素材层、分析层、缺口、SOP 状态和报告草稿。
